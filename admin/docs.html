API Documentation

Welcome to the WhatsApp Gateway API. This API allows you to programmatically send messages, images, documents, and manage groups via your connected WhatsApp session.

Ensure your session is Connected in the Dashboard before making API calls.

Authentication

All API requests must be authenticated using your unique API Key or a Bearer Token. You can generate this key in the Session Detail page.

Option 1: Bearer Token (Header)
Authorization: Bearer YOUR_TOKEN
Option 2: Query Parameter
?apikey=YOUR_API_KEY

Send Message

The primary endpoint to send messages of various types (Text, Image, Document).

Endpoint
POST /api/v1/messages?sessionId={sessionId}
cURL Example
curl -X POST "https://your-domain.com/api/v1/messages?sessionId=session1" \
-H "Authorization: Bearer YOUR_TOKEN" \
-H "Content-Type: application/json" \
-d '{
    "recipient_type": "individual",
    "to": "6281234567890",
    "type": "text",
    "text": {
        "body": "Hello World!"
    }
}'
Endpoint
POST /api/v1/messages?sessionId={sessionId}
cURL Example
curl -X POST "https://your-domain.com/api/v1/messages?sessionId=session1" \
-H "Authorization: Bearer YOUR_TOKEN" \
-H "Content-Type: application/json" \
-d '{
    "recipient_type": "individual",
    "to": "6281234567890",
    "type": "image",
    "image": {
        "link": "https://picsum.photos/200",
        "caption": "Check this out!"
    }
}'
Endpoint
POST /api/v1/messages?sessionId={sessionId}
cURL Example
curl -X POST "https://your-domain.com/api/v1/messages?sessionId=session1" \
-H "Authorization: Bearer YOUR_TOKEN" \
-H "Content-Type: application/json" \
-d '{
    "recipient_type": "individual",
    "to": "6281234567890",
    "type": "document",
    "document": {
        "link": "https://example.com/file.pdf",
        "mimetype": "application/pdf",
        "filename": "invoice.pdf"
    }
}'

You can send multiple messages (array of objects) in a single request.

curl -X POST "https://your-domain.com/api/v1/messages?sessionId=session1" \
-H "Authorization: Bearer YOUR_TOKEN" \
-H "Content-Type: application/json" \
-d '[
  {
    "recipient_type": "individual",
    "to": "6281234567890",
    "type": "text",
    "text": { "body": "First message" }
  },
  {
    "recipient_type": "individual",
    "to": "6281234567890",
    "type": "image",
    "image": { "link": "https://picsum.photos/200" }
  }
]'

Legacy Query Param API

This API style is compatible with older gateway formats. It supports both GET and POST.

GET/POST /api/v1/send-message
Parameter Type Description
apikey String *Required. Apikey your session.
mtype Enum *Required. Options: text, image, video, audio, audioconvert, document, sticker, stickerconvert
receiver String *Required. Receiver Phone Number with Country Code (e.g: 62812345678) or id group.
text String Required if mtype is text. Text Message.
url Url Required if mtype is image, video, audio, audioconvert, document, sticker, stickerconvert. url file. example : http://domain.com/example.jpg
filename String Optional. if you want a custom name of the document sent.
GET/POST /api/v1/fetch-group
Parameter Type Description
apikey String *Required. Apikey your session.
GET/POST /api/v1/fetch-member
Parameter Type Description
apikey String *Required. Apikey your session.
idgroup String *Required. ID group WhatsApp (e.g: 120163044481549146@g.us)
